Apriltag-Based Drone Tracking in Gazebo (ROS2 Simulation)

Project Overview

Developed an autonomous drone tracking system in a simulated Gazebo environment, where the drone follows a dynamically moving AprilTag using onboard camera vision and ROS2-based perception and control pipelines.

Features

  • Visual Detection: Integrated a camera on the drone to continuously detect AprilTags using the apriltag_ros package for real-time pose estimation.
  • Pose Tracking: Extracted the 6-DoF relative transform between the AprilTag and the drone using TF data to compute tracking errors.
  • Drone Control: Implemented a proportional control strategy that commands drone velocity based on the tag’s relative position.
  • Dynamic Tag Movement: Simulated the motion of the AprilTag along a predefined trajectory, requiring the drone to continuously adapt its path.
  • Simulation Setup: Utilized ROS2 with Ignition Gazebo for realistic drone and tag simulation, including camera plugins and dynamic models.

Tools & Frameworks

  • Ros2
  • Ignition Gazebo
  • apriltag_ros
  • MAVROS/PX4
  • TF2 for transform lookup
  • Rviz for visualization

Drone Localization using Multiple Apriltags (3D Trilateration)

Project Overview

Implemented a robust 3D localization system for a drone using multiple AprilTags with known world-frame positions. The system calculates the drone’s global position through trilateration and validates it against onboard IMU data.

Features

  • Fixed AprilTag Anchors: Placed multiple AprilTags at known positions in the simulation environment to act as reference landmarks.
  • Precise Trilateration: Estimated the drone’s 3D position by computing distances to each visible AprilTag and applying trilateration algorithms.
  • Pose Validation: Cross-verified the computed position with the drone’s onboard IMU and odometry data to assess localization accuracy.
  • Redundancy Handling: Supported partial tag visibility by dynamically selecting the most reliable subset of visible tags for trilateration.
  • Simulation and Testing: Used ROS2 and Gazebo for testing localization under various motion profiles and camera angles.

Tools & Frameworks

  • Ros2
  • Ignition Gazebo
  • apriltag_ros
  • Python/C++ trilateration logic
  • TF2 for transform lookup
  • IMU and odometry fusion for accuracy checks
  • Gazebo simulation with onboard camera plugin

Outcome

Enabled consistent and accurate global localization of the drone in outdoor environments, demonstrating the effectiveness of vision-based positioning using minimal infrastructure.
